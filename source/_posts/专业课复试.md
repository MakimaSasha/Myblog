## 计算机网络

### OSI 7 层模型

```markdown
1. 物理层：
	传输单位是比特
	任务是传输比特流
	包括一些机械的、电气等特性

2. 数据链路层：
	传输单位是帧，mtu：最大传输单元
	任务是将网络层传来的IP数据报组装成帧
	功能是组装帧、差错控制、流量控制、传输管理
	协议：SDLC、HDLC、PPP、STP、帧中继
	补充：
		PPP（Point-to-Point Protocol）：用来拨号连接，联系网关拨号

3. 网络层：
	传输单位是数据报
	任务是将数据单元从源传输道目的端
	功能是进行路由选择、流量控制、拥塞控制、差错控制、网际互连
	协议：IP、IPX、ICMP、IGMP、ARP、RARP、OSPF
	补充：
		IP：负责将数据报从源地址传输到目的地址
		ARP：将IP地址解析为MAC地址，在局域网中使用
		ICMP（Internet Control Message Protocol）：在IP网络上发送控制消息，用来进行网络故障排除和性能测试
		OSPF（Open Shortest Path First）：用于在IP网络中计算最短路径
4. 传输层：
	传输单位是TCP报文段或者UDP用户数据报
	任务负责两个进程（可以是同意主机也可以是不同主机）之间的通信
	功能实现端到端的连接，提供可靠的传输服务，流量控制、差错控制、服务质量、数据传输管理
	协议：TCP、UDP
	补充：
		虽然IP协议是面向无连接不可靠的，但是TCP对IP协议进行了补充，通过三次握手，对数据包进行分段、排序、确认、重传操作来保证数据包的可靠传输
	
5. 会话层：
	实现各个进程（可以是同意主机也可以是不同主机）之间进行会话即Session（计算机之间进行通信的过程），相当于在传输层上建立了更高层次的抽象和更加全面的服务
	任务为表示层实体或者用户进程建立连接，并在连接上有序地传输数据，这也叫会话，这个过程也叫建立同步（SYN）
	功能：建立、管理、终止会话，使用校验点恢复会话实现数据同步
	
6. 表示层：
	将数据表示为一种独立于机器的格式，便于在不同的系统进行传输和交换
	功能：数据格式化、加密与解密、压缩与解压缩、数据转换

7. 应用层：
	应用层是用户与网络的界面，不同的应用要求使用不同的协议
	协议：FTP、SMTP、HTTP、HTTPS、DNS
	补充：
        HTTP协议是明文传输，数据容易被窃听和篡改，而HTTPS协议则使用SSL/TLS协议对数据进行加密传输，保证数据的安全性。
        TLS（Transport Layer Security）是在SSL（Secure Sockets Layer）基础上进行改进和完善来提高安全性和可靠性，它支持更多的加密算法，握手协议也不同TLS更加安全
        SSL实现加密的方法：浏览器把公钥发给服务器，服务器用公钥加密，然后将加密数据发给浏览器，浏览器用私钥解密
```

### TCP/IP四层模型

```markdown
数据链路层：提供链路的管理、错误检测
网络层：负责网络中数据包的传送
传输层：实现数据传输和数据共享
应用层：接收处理来自传输层的数据或者按不同应用要求与方式将数据传输至传输层
```



### TCP和UDP协议的区别

```markdown
面向连接：通信双方必须先建立连接，分配相应的资源（如：缓冲区），以保证通信能正常进行，传输结束后释放连接和所占用的资源，流程为建立连接->数据传输->连接释放，如TCP协议
无连接：不用建立连接，直接发送数据，尽最大努力交付，不保证通信的可靠性，如IP、UDP协议

可靠：指网络具有纠错、检错、应答机制，能够保证数据的正确性
不可靠：尽量正确、可靠地传输数据，不保证数据正确、可靠地传送道目的地，尽力而为的服务

	TCP是面向连接的可靠传输协议、传输速率慢、开销大（报文段头部长），不提供广播和多播服务
	UDP是无连接的不可靠传输协议、传输数据较快、开销小（报文段头部短），时延小，传输后无需确认
```

### 虚电路网络&数据报网络

```markdown
虚电路网络：通过建立虚拟的连接来保证数据传输的可靠性和流量控制，还可以实现避免网络拥塞
	建立连接：建立虚电路连接，设置虚电路号、连接参数等，建立连接后虚电路网络会分配一些带宽、缓存给这条虚电路
	传输数据：数据包会被标记上虚电路号，通过虚电路传输，保证数据包按照连接建立的顺序到达目的地
	拆除连接：释放资源

数据报网络：不需要连接，直接将数据分割成若干个数据报，每个数据报包含目的地址、源地址、序列号、校验和，通过网络中的路由器、交换机进行转发，到达目的地后进行重组还原为原始数据
	分组：将数据报分割成若干个数据包
	路由选择：根据目的地址选择最优的路径，将数据包转发到下一个路由器或者交换机
	转发：根据转发表将数据报转发到正确的端口
	重组：在目的地将数据报进行重组
1. 两者的共同点
	都是分组交换的方式

两者的区别：
	虚电路面向连接，数据报是无连接的
	虚电路的数据包的传输顺序和路径都是确定好的，数据报的数据包传输路径和传输数据都是动态确定的
	虚电路传输延迟小，数据报传输延迟大
	虚电路传输可靠，数据报可靠性低可能出现丢失或错误

补充：数据通信方式，网络层
	电路交换：建立专用的物理连接传输数据，独占线路，数据传输速度稳定时延小，实时性强，交换设备成本较低，但线路利用率低、电路持续时间长、通讯效率低，使用于信息量大、长报文，经常使用的固定用户之间的通信
	
	报文交换：基于存储转发技术，将用户的报文存储在交换机的存储器中，当需要输出的电路空闲时再发送报文，中继电路利用率高，但传输时延打、占用大量交换机内存和外存。适用于传输的报文较短、实时性要求低的网络，如公用电报网
	
	分组交换：实质上是在存储转发基础上发展的，具有电路交换和报文交换的优点。采用动态复用技术传输按一定长度分割的分组。把数据暂存在交换机的存储器中，然后再转发。利用率高于电路交换，且比报文交换传输时延小、交互性好
	
	复用与分用：
		信道复用技术：频分复用、时分复用、波分复用、统计时分
```

### NAT

```markdown
1. NAT（Network Address Translation，网络地址转换）
	任务：将私有网络中的IP地址转换为公共网络中的IP地址，以实现私有网络与公共网络之间的通信，通常由路由器实现
	作用：
		1. 提高网络安全性：NAT技术可以隐藏私有网络中的IP地址，使得公共网络中的攻击者无法直接访问私有网络中的设备，从而提高网络安全性。
        2. 减少IP地址的使用：由于IPv4地址空间有限，NAT技术可以将多个私有IP地址映射到同一个公共IP地址上，从而减少IP地址的使用，在⼀定程度上，也缓解了IPV4地址匮乏的问题。
        3. 允许多个设备共享同一个公共IP地址：由于IPv4地址空间有限，NAT技术可以让多个设备共享同一个公共IP地址，从而节约IP地址的使用。
        4. 实现端口映射：NAT技术可以将公共网络中的端口映射到私有网络中的端口，以便在公共网络中访问私有网络中的服务。


2. 补充：NAT打洞
        1. 设备A和设备B都向外网发送一个UDP数据包，这个数据包的目的地址和端口为对方设备的公网IP地址和随机端口。这个数据包的作用是让NAT路由器记录下A和B的公网IP地址和端口，并将这个映射关系保存在NAT路由器中。
        2. 设备A和设备B都向对方的公网IP地址和端口发送一个UDP数据包，这个数据包的作用是让NAT路由器将对方设备的公网IP地址和端口映射到A和B的私有IP地址和端口上。由于NAT路由器已经记录了A和B的公网IP地址和端口，因此这个数据包可以穿透NAT路由器，并将对方设备的映射关系保存在A和B的NAT路由器中。
        3. 设备A和设备B通过对方的公网IP地址和端口互相发送UDP数据包，测试是否能够直接通信。如果可以直接通信，则说明NAT打洞成功，两个设备之间已经建立了直接的连接。

    需要注意的是，NAT打洞技术的实现涉及到网络协议和底层实现，需要使用相应的网络库和操作系统接口。在实际应用中，需要根据具体的需求和网络环境，使用相应的网络库和操作系统接口实现NAT打洞技术。
```

### HOL阻塞

```markdown
	在网络传输中，由于单个数据包在传输过程中出现错误或者被丢失，导致该数据包之后的所有数据包都被阻塞，无法传输的现象，通常发生在TCP协议中，因为当TCP传输的数据包出现错误和丢失时，TCP会等待该数据包被重传后，才继续传输后面的数据包，TCP协议采用流量控制和拥塞控制机制，当出现网络阻塞时，会降低发送速率。
	
	避免方法：
		1. 使用UDP传输数据
		2. 使用多个TCP链接进行数据传输，将数据分山在多个连接中传输
		3. 采用拥塞控制算法，如：TCP Vegas，TCP BBR
```

### 集线器、中继器、路由器的区别

```markdown
1. 物理层设备，进行互联、扩展局域网的连接设备：
        中继器：连接连个速率相同且数据链路层协议也相同的网段，作用是消除数字信号的衰减和失真，即用来信号再生
        集线器：相当于多接口的中继器，将多个节点连成一个共享式局域网，即用来将多个设备连接到一个局域网上，广播数据（加强版的中继器）
	
2. 网络层设备，输入端和输出端是不同的子网
		路由器：主要功能进行路由选择、转发数据包、连接不同的网络
		交换机：将多个设备连接到同一个局域网内，根据MAC地址将数据直接转发给目标，还可以进行流量控制（相当于是加强版集线器）
```

### 在一个TCP的通信中，现在主机A向主机B发送一个序列号为100的两字节的报文，他问B是否一定会传一个102的确认号回去啊？是否一定大于102或者是一定小于102？

```markdown
不一定，有可能A发送的字节都丢失，B返回的序列号一定小于或者等于102，（序列号用来标识TCP数据流中的每一个字节），B返回的确认号表示接收方期待的下一个数据包的序列号，也就是当这两个字节正确接收后，B会返回102
```

### TCP连接

```markdown
建立连接三次握手：
	1. 客户端向服务器发送同步请求（SYN），表示客户端想要建立连接，并给出初始序列号
	2. 服务器收到SYN请求后，向客户端发送确认请求（ACK），表示服务器收到了客户端的SYN请求，并告诉客户端服务器的初始序列号
	3. 客户端收到服务器的ACK后，向服务器发送ACK表示客户端收到服务器的确认请求，并确认服务器的初始序列号，成功建立连接

关闭连接四次挥手：
	1. 主动关闭方发送FIN给被动关闭方，表示自己没有数据要发送，但此时还可以接收数据
	2. 被动关闭方接收到FIN后，发送ACK，表示已经收到了关闭请求
	3. 被动关闭方发送自己的FIN，表示自己也没有数据要发送了，即双方可以关闭连接了
	4. 主动关闭方接收到被动关闭方的FIN后，发送ACK，表示已经收到了关闭请求。被动关闭方接收到ACK后关闭自己的连接，主动关闭方在发送ACK后等待2MSL后关闭自己的连接
```

### 数据带宽是10Mb/s，一个用戶传数据需要占1Mb/s，问电路交换可以有几个用戶一起传输数据？在分组交换下，为什么10个以下用戶没有排队时延？为什么11个以上用戶有排队时延

```markdown
电路交换可以有10个用户一起传输数据，因为一个用户占用1Mb/s，10个用户正好占满

分组交换时，将报文划分为小的分组，每个分组独立地选择传输路径，10个一下用户带宽小于10Mb/s，通过独立选择传输路径，可以避开繁忙路径，使得没有排队时延.11个以上用户时所需带宽大于10Mb/s，哪怕选择独立路径仍会出现排队情况
```

### GBN协议

```markdown
	即回退N帧，数据链路层协议，用于可靠传输，基于窗口的协议，允许发送方连续发送多个数据包，接收方按序接收数据包，并发送确认消息，GBN协议采用了滑动窗口机制，确认一个帧后，滑动一下，发送窗口前面的都是已经确认的帧，窗口后面的都是还没有发送的帧，所以接收方不会接收到窗口以外的确认号
```

### 分组转发，时延

```markdown
时延包括：发送时延、传播时延、处理时延、排队时延
其中起决定因素的是排队时延，主要是当网络流量过大，路由器无法一次性处理完所有要发送的分组时，当路由器的队列满了后，后面再接收到的分组就会被抛弃，即出现丢包现象
IP协议是面向无连接的不可靠协议，不会重传，但如果上层使用的是TCP协议，TCP是面向连接的可靠协议，有差错控制会重传
```

## 编译原理

### 编译程序的结构（工作原理）

```markdown
源程序->词法分析->语法分析->语义分析->中间代码生成->代码优化->目标代码生成->目标代码

   1. 词法分析：对源程序的字符进行扫描和分解，识别单词（记号）
   2. 语法分析：层次结构分析，建立语法树
   3. 语义分析：检测结构正确的句子是否语义合法，合法则翻译成中间代码
   4. 代码优化：对中间代码进行优化
   5. 目标代码生成：将中间代码变换成特定机器上的目标代码
   6. 目标代码：
        1.绝对指令代码：能够立即执行的机器语言代码，所有的地址都已经定位
        2.可重定位指令代码：机器语言模块，需要与运行程序链接才能执行
        3.汇编指令代码：需要汇编程序转换成可执行机器语言
```

### 基础概念

```markdown
1. 文法：以有穷的集合来刻画无穷的集合的一个工具

2. 字母表：元素的非空集合，其中的元素又称为符号，因此字母表也称为符号集，比如{1,2}就可以是一个字母表，用其中的元素组成的序列就叫符号串比如111或者1122随便组成有穷序列就是符号串

3. 符号串：由字母表中的符号组成的任何有穷序列称为符号串
	闭包就是字母表的符号组成的符号串的有穷序列比如：{11,112,11112}这个就是闭包，正闭包就是闭包中不含空（空就是字母表的的0次幂）

4. 产生式：A->a读作A定义为a，这就是一条关于A的产生式

*文法的定义* ------ 四元组G = (VN, VT, P, S)
	VN:非终结符集，通常用大写字母表示，S通常表示开始符号
	VT: 终结符集，通常用小写字母，运算符，数字表示
	P：产生式集，如A->b
	S：识别符或开始符，最开始的那条产生式的左部
	**注意这些都只是定义，在实际的编程语言中，终结符这些东西是不太一样的，得看编程语言怎么去定义的
	例子：G = ({S,A,B}, {0,1},p,s)
	
	

*语言的定义*------ L(G[s]) = {x | s=> +x 且 x 属于 VT*}

补充1：
	从文法的开始符号出发，反复连续地使用产生式，对非终结符进行展开和替换，最终得到一个仅有终结符构成的符号串，推导的每一步都是一个直接推导
	1.直接推导：只使用1次规则推导出右部，就是变换一次记为：A->a
	2.推导：使用至少一次规则推导出右部 记为：a0 => +a1	这个+号就是推导的标志
	3.广义推导：经过0步或者若干步推导出右部 记为：a0 => *a1

补充2：
	最左推导：就是每次替换时优先选择最左边的非终结符进行替换 比如: G[E]: E->A+B A->i B->j 则推导出句型E->i+j可以使用最左推导先替换A，E->i+B，然后替换B,E->i+j
	最右推导：又称规范推导，由规范推导所得的句型称为规范句型，道理类似最左推导至少先替换B然后替换A
	
	
	
```

### 编程语言绑定的概念，静态绑定和动态绑定的区别

```markdown
概念：
	编程语言绑定是指程序中的名称（变量、函数、类等）与其对应的内存地址进行关联的过程
静态绑定：
	编译时进行名称和内存地址的关联，编译器会根据声明的类型确定名称对应的内存地址，在程序运行之前就已经确定。如c/c++。静态绑定要实现多态需要通过虚函数（c++中使用virtual来修饰）来实现，就是子类对父类函数的重写
	C++中的虚函数：在基类中在函数的声明前面加上virtual关键字，就可以让它在这个基类的派生类中被重新定义（之所以被称为虚函数就是因为它在运行时才确定具体调用哪个函数）
动态绑定：
	程序中的名称和内存地址的关联是在程序运行时才确定的，程序可以根据运行时的实际情况进行绑定，如：python、Ruby。动态绑定可以在运行时根据对象的类型来调用对应的方法。
```

### 自动机

```markdown
	编译原理中的自动机是一种用于识别和处理输入符号串的计算模型。它可以根据输入符号的序列转移状态，并根据状态最终接受或拒绝输入

	常见的自动机包括
        有限状态自动机（FSM）
        正则表达式自动机
        上下文无关文法自动机等

自动机在编译原理中被广泛应用于词法分析、语法分析和语义分析等方面，实质就是用来识别编程语言的语法是否规范。
编译器将源代码转换为一种中间形式，然后自动机进行语法分析和语义分析，最终生成目标代码
```

### 文法乔姆斯基体系

```markdown
语法就是使用编程语言时需要遵守的规则，比如什么是合法的程序结构，语法结构等
文法用来描述一种语言的结构和形式的规则体系

乔姆斯基体系就是根据语法规则的形式化程度，将文法分为四个层次：
	0型文法：	递归可枚举语言		最复杂		图灵机				例如：A0->A0，		其中输入输出没有限制
	1型文法：	上下文有关语言				线性有界非确定图灵机	  例如：1A1->101011	要求：左边小于右边
	2型文法：	上下文无关语言				非确定下推自动机	例如：G[S]: S->01 S->0S1	要求：在1型文法的要求上，还需要左边必须得是一个非终结符，这样就使得非终结符总是可以被替换，所以不用考虑其前后的字符，因此叫上下文无关语言
	3型文法：	正则语言		最简单		有限状态自动机		例如：G[S]: S->0A|0 A->1B|B B->1|0	要求：在二型文法的基础上，还要求产生式满足右线性或者左线性
	
补充：
	1. 0型文法又叫无限制文法就是说输入和输出可以是任意字符串，用来描述图灵机的行为
	2. 右线性文法（Right-linear Grammar）是指产生式中只包含右线性的规则，即每个产生式的右部只包含一个终结符或一个终结符和一个非终结符的组合。例如：A -> aB 或 A -> a。
	3. 左线性文法（Left-linear Grammar）是指产生式中只包含左线性的规则，即每个产生式的右部只包含一个终结符或一个非终结符和一个终结符的组合。例如：A -> Ba 或 A -> a。
```

### 简单介绍slr(1)

```markdown
1. LR是一种自底向上的语法分析算法，全程为Left-to-right，能够处理大多数上下文无关文法，LR(K):L表示从左往右，R表示反向构造出一个最右推导，K表示向前K个字符，缺省为1

2. SLR是一种自底向上的语法分析算法，Simple LR，即简单LR。SLR(1)在LR(0)的基础上，使用向前看符号进行规约，其中1表示每一个规约项在分析时只需要向前看一个符号，SLR(1)相比与LR(1)的分析表更小，分析速度更快，但文法限制更多，只能处理一些简单的语法结构.如果想处理更加复杂的语法结构，可以使用LR(k)算法，k表示向前看的符号数量

可以理解成SLR是LR的一种变体
```

### 递归下降分析法

```markdown
	递归下降分析法是一种语法分析方法，它从文法的起始符号开始，按照产生式的结构逐步分析待分析的语句。在分析过程中，对于文法的每个非终结符号，都会编写一个对应的子程序，用来识别该非终结符号所代表的语法范畴。这些子程序会递归调用其他子程序，直到完成对整个语句的分析。也就是说，递归下降分析法是一种基于递归的自上而下的分析方法。
```

### 短语、句子、句柄、素短语

```markdown
1. 句型：通过S经过一次或多次推导得到x，则称x为G[S]的句型
2. 子树：由一非末端结点（就是分支结点）连同所有分支组成的部分，子树里面如果也有非末端节点则这个非末端结点....递归定义就完事了
3. 直接子树：树高为2的子树
4. 短语：子树的末端结点形成的字符串，从左到右，把末端节点都取出来组成的字符串
5. 直接短语：首先它是短语，其次A=>b，则b下面不能有分支，实际上就是直接子树的叶子组成的字符串
6. 素短语：至少包含一个终结符且不含更小素短语的短语
7. 句柄：一个句型的最左直接短语，就是先把直接子树画出来
```
